这份开发文档旨在帮助在两周内快速构建一个具备高技术密度的 **AI Travel Agent**。我们忽略繁琐的 UI 细节，将核心精力集中在企业看重的 **Agent 逻辑控制、RAG 检索增强和多模态输出**上。

---

## 项目名称：GeoTrave (智能多代理旅游规划引擎)

### 一、 项目主要技术栈
* **核心框架：** **LangGraph** (首选，用于构建复杂的循环 Agent 状态机) 或 **CrewAI**。
* **大模型：** **GPT-4o / Claude 3.5 Sonnet** (主要逻辑与 VLM 图片识别)。
* **实时搜索：** **Tavily API / Serper API** (专为 Agent 设计的搜索引擎)。
* **数据存储/RAG：** **ChromaDB** 或 **FAISS** (向量数据库)，用于存储离线攻略或用户偏好。
* **辅助工具：** **OpenPyXL** (生成 Excel 行程表)、**Static Maps API** (地理位置标注)。
* **前端演示：** **Streamlit** (快速搭建 Web 交互界面)。

---

## 二、 项目开发目标
1.  **展示 Agent 决策能力：** 能够根据用户模糊需求（如“想去关西吃美食，预算 5000”）自主拆解任务。
2.  **解决实时性问题：** 能够通过搜索工具获取最新的机票价格和景区开放情况。
3.  **展示工程化思维：** 实现结构化输出（JSON/Markdown）和长短期记ChromaDB忆管理。
4.  **证明技术深度：** 包含 AutoPrompt 优化和 RAG 检索链路。

---

## 三、 项目核心功能
* **多智能体协作：**
    * *Planner Agent:* 负责拆解任务（机、酒、景、食）。
    * *Searcher Agent:* 负责实时抓取网络信息并进行初步筛选。
    * *Optimizer Agent:* 负责根据预算和地理位置，调用算法优化路线。
* **实时机票/酒店比价逻辑：** 模拟调用接口获取低价建议。
* **结构化行程导出：** 自动生成生动的 Markdown 手册及 Excel 时间轴表格。
* **可视化路线：** 利用经纬度生成静态地图标记点序列。

---

## 四、 开发详细步骤 (两周冲刺版)

### 第一阶段：架构与基础链路 (第 1-4 天)
1.  **环境搭建：** 配置 Python 环境，集成 LangChain/LangGraph，获取 OpenAI 和 Tavily 的 API Key。
2.  **定义 State 结构：** 在 LangGraph 中定义全局状态 `AgentState`，用于在不同 Agent 间传递机票信息、景点列表和用户偏好。
3.  **实现核心 Node：**
    * 编写 `Search_Node`：调用 Tavily 搜索特定城市的“Top 10 美食”或“今日票价”。
    * 编写 `Router`：判断当前信息是否足够，决定继续搜索还是开始规划。

### 第二阶段：RAG 与工具集成 (第 5-8 天)
1.  **本地知识库 (RAG)：** 将一些高质量的旅游攻略 PDF 或 Wiki 存入 ChromaDB，展示 Agent 如何结合“网络实时信息”与“本地专业攻略”。
2.  **Tool Calling 开发：**
    * 编写 `Calculator_Tool`：精确计算机票总价及人均开销。
    * 编写 `Map_Formatter_Tool`：将文本地址转化为经纬度（可调用 Google Geocoding API）。
3.  **AutoPrompt 优化：** 编写一个 `System_Prompt_Manager`，根据用户输入（如：穷游、亲子、特种兵）自动注入不同的约束条件。

### 第三阶段：输出优化与可视化 (第 9-12 天)
1.  **模板引擎：** 编写 Jinja2 模板，将 Agent 的 JSON 输出渲染成精美的 Markdown 报告。
2.  **地图集成：** 实现一个函数，将路径点转化为 HTML 形式的地图组件（使用 Python 的 `folium` 库非常简单）。
3.  **异常处理：** 加入“反思（Reflection）”机制，如果 Agent 生成的行程在地理上不合逻辑（例如上午在东京下午在大阪），触发重新规划。

### 第四阶段：演示准备 (第 13-14 天)
1.  **Streamlit UI：** 编写一个简单的侧边栏用于输入参数，中间显示 Agent 的思考过程（Thought Process）和最终行程。
2.  **文档撰写：** 编写 README.md，重点画出 **Agent 逻辑拓扑图**。
3.  **录屏与部署：** 录制 Agent 如何处理复杂矛盾需求（如“想省钱但又要住五星级酒店”）的思考逻辑。
